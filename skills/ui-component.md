# ui-component skill

Given a component name, description, and props:

1. Check `context/ui-registry.md` to avoid duplication.
2. Use `context/ui-tokens.md` for all design values (colors, spacing, typography).
3. Implement the component with all states: loading, empty, error, disabled.
4. Follow `context/ui-rules.md` for layout and behavior.
5. Include TypeScript interfaces and JSDoc comments.
6. Write a matching Storybook story or test.
7. Register the new component in `context/ui-registry.md` with import path, props, and example.
